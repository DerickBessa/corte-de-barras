📌 Resumo do Problema

O problema de Corte de Barras (Rod Cutting) é clássico em algoritmos:

Dada uma barra de comprimento n e uma tabela de preços p[i] para cada pedaço de tamanho i, encontre a melhor forma de cortar a barra para maximizar a receita total.

  1. Criação do repositório GitHub
    O repositório DerickBessa/corte-de-barras foi criado publicamente para versionar e entregar o projeto.
  2. Implementação dos 3 algoritmos (src/algorithms.py)
    Três abordagens diferentes foram implementadas para resolver o mesmo problema:
    • Guloso — escolhe localmente o melhor pedaço (maior preço por unidade de tamanho). Rápido, mas não garante o ótimo.
    • Força Bruta — testa recursivamente todas as formas de cortar a barra. Exponencial O(2ⁿ) — inviável para n > 25.
    • PD (Memoização) — recursivo com memoização, O(n²). Garante o ótimo e retorna os cortes.
  3. Geração de entradas de teste (src/generators.py)
    Função gerar_tabela_precos(n, seed) cria tabelas de preço com ruído aleatório para simular cenários realistas e dar chance do guloso errar.
  4. Verificação de corretude (src/verify.py)
    Duas verificações cruciais:
    1. FB vs PD — nos tamanhos pequenos (n = 5 a 21), a Força Bruta e a PD devem retornar o mesmo valor máximo. Isso valida que a PD está correta.
    2. Guloso vs PD — em todos os tamanhos, a PD deve ser sempre ≥ que o Guloso. A diferença média e o desvio padrão são calculados e reportados.
  5. Exemplo concreto onde o Guloso erra
    Uma tabela de preços específica foi criada para demonstrar visualmente que o Guloso pode falhar. No exemplo, o Guloso obtém um valor menor que a PD.
  6. Benchmark de tempo (src/benchmark.py)
    Para cada tamanho de barra, o experimento gera 10 tabelas de preço distintas, executa cada algoritmo nessas 10 entradas, e tira a média do tempo e o desvio padrão. Os resultados são salvos em results/tempos.csv.
  7. Geração de gráficos
    Três gráficos são gerados automaticamente:
    • Gráfico 1: Os 3 algoritmos juntos (tamanhos pequenos) — mostra o crescimento exponencial da FB
    • Gráfico 2: Guloso vs PD (tamanhos grandes) — mostra O(n) vs O(n²)
    • Gráfico 3: Diferença Guloso vs PD (barra de erro) — mostra o quanto o guloso perde
  8. Relatório no Jupyter Notebook (notebooks/analise.ipynb)
    Discussão completa dos resultados com texto explicativo, código executável e os gráficos embutidos.
  9. Esta página HTML
    Documentação visual do experimento com abas para navegação, incluindo o guia do vídeo de apresentação.

Os 3 Algoritmos em Detalhe

1. Guloso Ingênuo — O(n²) na prática

A cada passo, olha o tamanho restante e escolhe o pedaço com melhor preço por unidade de tamanho.

def corte_guloso(n, precos):
    restante = n
    cortes = []
    while restante > 0:
        melhor_razao = -1
        melhor_tam = 1
        for t in range(1, restante + 1):
            razao = precos[t-1] / t
            if razao > melhor_razao:
                melhor_razao = razao
                melhor_tam = t
        cortes.append(melhor_tam)
        restante -= melhor_tam
    return sum(precos[t-1] for t in cortes), cortes

⚠ Não garante o ótimo — decisões locais podem levar a um resultado sub-ótimo.

2. Força Bruta — O(2ⁿ)

Testa recursivamente todas as partições possíveis da barra. Para n=20, são ~500 mil combinações; para n=30, mais de 500 milhões.

def corte_forca_bruta(n, precos):
    if n == 0: return 0, []
    melhor_valor = -inf
    melhor_cortes = []
    for i in range(1, n + 1):
        v_resto, c_resto = corte_forca_bruta(n - i, precos)
        valor = precos[i-1] + v_resto
        if valor > melhor_valor:
            melhor_valor = valor
            melhor_cortes = [i] + c_resto
    return melhor_valor, melhor_cortes

⚠ Inviável para n > 25 — usada apenas como referencial teórico e verificação de corretude.

3. PD com Memoização — O(n²) — SOLUÇÃO RECOMENDADA

Recursivo com memoização: cada subproblema é resolvido uma única vez e reaproveitado. Além do valor máximo, guarda o primeiro corte escolhido para reconstruir a solução.

def corte_pd_memo(n, precos):
    memo_valor = {}
    memo_corte = {}

    def f(m):
        if m == 0: return 0
        if m in memo_valor: return memo_valor[m]
        melhor = -inf
        for i in range(1, m + 1):
            valor = precos[i-1] + f(m - i)
            if valor > melhor:
                melhor = valor
                melhor_primeiro = i
        memo_valor[m] = melhor
        memo_corte[m] = melhor_primeiro
        return melhor

    valor_total = f(n)
    cortes = []
    resto = n
    while resto > 0:
        c = memo_corte[resto]
        cortes.append(c)
        resto -= c
    return valor_total, cortes

✓ Garante o ótimo — solução exata com complexidade polinomial.

Conclusão Principal

A Programação Dinâmica com memoização é a melhor abordagem para o problema de Corte de Barras: solução exata, complexidade O(n²), escala para entradas grandes. O Guloso é mais rápido mas não garante o ótimo. A Força Bruta é apenas um conceito didático.
