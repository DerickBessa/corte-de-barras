O Problema do Corte de Barras
Rod Cutting — Análise de Algoritmos · IFCE
Definição

Dada uma barra de comprimento n e uma tabela de preços p[i] para cada pedaço de tamanho i, encontrar a melhor forma de cortar a barra para maximizar a receita total.

Partições possíveis: 2n−1 — crescimento exponencial

Os 3 Algoritmos
Guloso · Força Bruta · Programação Dinâmica

🟠 Guloso

Escolhe o pedaço com melhor preço por unidade a cada passo. 

while restante > 0:
    melhor_razao = -1
    for t in range(1, restante+1):
        razao = precos[t-1] / t
        if razao > melhor_razao:
            melhor_tam = t
    cortes.append(melhor_tam)
    restante -= melhor_tam
O(n²) Não garante o ótimo

🔴 Força Bruta

Testa recursivamente todas as partições possíveis. 

def f(n):
    if n == 0: return 0
    for i in range(1, n+1):
        valor = precos[i-1] + f(n-i)
        if valor > melhor:
            melhor = valor
    return melhor
O(2ⁿ) Inviável n>25

🟢 PD com Memoização

Resolve cada subproblema uma vez, reaproveita resultados. 

memo_valor = {}
memo_corte = {}

def f(m):
    if m in memo_valor:
        return memo_valor[m]
    for i in range(1, m+1):
        valor = precos[i-1] + f(m-i)
        if valor > melhor:
            melhor_valor = valor
            melhor_corte = i
    memo_valor[m] = melhor_valor
    memo_corte[m] = melhor_corte
    return melhor_valor
O(n²) ✅ Garante o ótimo
Diferença chave: Guloso e PD têm a mesma complexidade O(n²) na notação teórica, mas o Guloso é linear na prática. Já a Força Bruta é O(2ⁿ) — impraticável.
